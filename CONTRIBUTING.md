# Contributing Guide

Thank you for participating in the **CSK Major Hackathon – Mobile Development Track**.

This document explains how to submit your solutions for the daily challenges and final project. Following these guidelines helps keep the repository organized and makes it easier for mentors to review submissions.

---

## Participation Workflow

1. Fork the repository to your own GitHub account.
2. Clone your fork to your local machine.
3. Complete the daily challenge.
4. Add your solution to the correct submission folder.
5. Commit and push your changes.
6. Open a Pull Request to this repository.

---

## Repository Setup

Fork the repository and clone it locally.

Example:

```
git clone https://github.com/your-username/csk-mobile-hackathon.git
cd csk-mobile-hackathon
```

Create a new branch for your work.

```
git checkout -b day-04-flatlist
```

Work on the challenge and commit your solution.

---

## Submission Folder Structure

All solutions must be placed inside the **submissions** directory.

Example structure:

```
submissions/
  day-01/
  day-02/
  day-03/
  day-04/
```

Inside the correct day folder, create a folder with your name.

Example:

```
submissions/day-04/reuben-flatlist/
```

Your solution folder should contain the files necessary to run your project.

Example:

```
reuben-flatlist/
  App.js
  components/
  screens/
  assets/
```

Keep your code organized and easy to understand.

---

## Pull Request Guidelines

When submitting your work:

* Submit **one Pull Request per challenge**
* Ensure your code runs without errors
* Place files in the correct folder
* Use clear commit messages

Example Pull Request title:

```
Day 04 Challenge — FlatList Implementation
```

Example Pull Request description:

```
Challenge: Day 04 - FlatList

What I built:
A contacts list using FlatList.

Extra features:
Added avatars and custom styling.
```

Screenshots or screen recordings are optional but encouraged.

---

## Code Quality Guidelines

To keep submissions readable and maintainable:

* Use clear and descriptive variable names
* Keep components small and reusable
* Avoid unnecessary dependencies
* Follow basic React Native project structure

Example structure:

```
components/
screens/
assets/
```

---

## Challenge Rules

Participants should:

* Attempt the challenge independently
* Use the provided starter code if available
* Ask questions when stuck
* Respect other participants and mentors

Collaboration and discussion are encouraged, but each participant should submit their own implementation.

---

## Final Project

At the end of the hackathon, participants will build a **complete mobile application** using the concepts learned during the daily challenges.

Final project submissions should be placed inside:

```
submissions/final-projects/
```

Example:

```
submissions/final-projects/reuben-campus-events-app/
```

Projects should include:

* Multiple screens
* Navigation
* Reusable components
* Clean styling
* Lists using FlatList
* Either API data or local storage

---

## Need Help?

If you encounter issues:

* Review the challenge instructions
* Ask questions in the hackathon discussion channel
* Reach out to mentors for guidance

The goal of this hackathon is to learn by building and experimenting.

---

Thank you for contributing and participating in the CSK Major Mobile Hackathon.
